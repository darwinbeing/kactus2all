Register editor

Register editor is used to edit the details of a single register by assigning bit fields to a containing register. A bit field may contain just one bit, the whole register or something in between.

Field contains a name and a textual description. Offset describes the starting bit of the field within the containing register. Width specifies how many bits are included in the field. Volatile value of true indicates that there is no guarantee what a read operation will return because the register may change its value without write operations i.e. as a result of an interrupt.

Access column specifies the accessibility of the field, the possible values are:

  • Read-write
  • Read-only
  • Write-only
  • Read-writeOnce
  • writeOnce

The modified write value describes how the data in the field is manipulated on a write operation. The possible values are:
oneToClear
- indicates that each data bit of one written to a field will set the corresponding bit to zero.
oneToSet
- indicates that each data bit of one written to a field will set the corresponding bit to one.
oneToToggle
- indicates that each data bit of one written to a field will toggle the corresponding bit.
zeroToClear
- indicates that each data bit of zero written to a field will set the corresponding bit to zero.
zeroToSet
- indicates that each data bit of zero written to a field will set the corresponding bit to one.
zeroToToggle
- indicates that each data bit of zero written will toggle the corresponding bit.
Clear
- indicates that each write operation will clear all bits in the field.
Set
- indicates that each write operation will set all bits in the field.
Modify
- indicates that after a write operation all bits in the field can be modified.
Empty setting
- indicates that the value written to a field is the value stored in the field

Read action specifies the action that happens to the field after a read operation, the possible values are:
Clear
- indicates that all bits in the field are cleared after a read operation.
Set
- indicates that all bits in the field are set after a read operation.
Modify
- indicates that the bits in the field are modified in some way after a read operation.
Empty setting
- indicates that field is not modified after a read operation.

Testable specifies if the field is testable by an automated register test. Test constraint specifies the constraints for the automated tests for the field, the possible values are:
unConstrained
- indicates that there are no constraints for the written or read data. This is the default setting.
Restore
- indicates that the fields value must be restored to its original value before accessing another register.
writeAsRead
- indicates that the data written to a field must be same that was read previously from the field.
readOnly
- indicates that the field can only be read.
