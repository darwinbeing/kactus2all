Address block editor

Address block editor is used to edit the details of an address block by defining registers to it. Each row in the editor specifies a single register within the containing address block. The name, display name and a description contain identification information of an address block.
Usage specifies the usage of the address block (mandatory).
Base address specifies the starting address for the address block, expressed as address unit bits (mandatory, can be given as an expression).
Range specifies the size of the block, expressed as address unit bits (mandatory, can be given as an expression).
Width is the data width in bits of a row in the address block (mandatory, can be given as an expression).

Access specifies the accessability of the block.

  • read-write. Both read and write transactions may have an effect on this address block.
  • read-only. Only read transactions are allowed in this register.
  • write-only. Only write transactions are allowed in this register.
  • read-writeOnce. Read actions and the first write action may have an effect on this address block.
  • writeOnce. Only the first write action affects the contents of this address block.

Volatile=true indicates that the stored value may change without the master's write operation.

Registers table

The register table enables the user to define registers for the address block. Each register has a name and a textual description.
Offset specifies the location of the register from the start of the containing address block expressed as number of addressing units. The actual address of the register is the sum of address block's base addr and register offset. E.g. if base addr=0x100 and reg has offset=0x4, then register's address=0x104.
Size defines the number of data bits the register contains. It must be less or equal to Width of the address space.
Dimension assigns an unbounded dimension to the register. Its semantics are the same as the C language standard for the layout of memory in multidimensional arrays.
Volatile=true indicates that register value may change without a write operation to it, i.e. by an interrupt event.

Access column specifies the accessibility of the register, the possible values are:

  • read-write. Both read and write transactions may have an effect on this register.
  • read-only. Only read transactions are allowed in this register.
  • write-only. Only write transactions are allowed in this register.
  • read-writeOnce. Both read and write transactions may have an effect on this register. Only the first write transaction, after an event that caused the reset value of the register to be loaded, may affet the contents of the register, and read transactions return a value related to the values in the register.
  • writeOnce. Only the first write transaction affects the contents of the register.

Reset value defines register's value on reset and Reset mask defines the bits of the register that have a known reset value. Bit value of 1 means that the corresponding bit has a known reset value, whereas 0 means that the value is unknown. Both use SystemVerilog syntax for binary values e.g. 'b1011, 4'b1011, and 4'b10_11 are all valid values. The number of bits must equal the register size.
