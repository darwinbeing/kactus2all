File editor

File editor can be used to edit the details of a single file within a file set.

Name field contains the relative file path from the containing component's xml-file to this file. File types list specifies the type of the file. File type can be used i.e. when specifying build commands for a group of files.

General options define 5 optional things. Logical name specifies e.g. a VHDL library where the VHDL-file will be compiled to. If it is "only used as default", it can be overridden by another process. If check box is unticked, then this logical name cannot be overrriden.
The check boxes is include file and contains external declarations can be used to specify the file is an include file and that the file contains external declarations and is needed by other files in this file set. Description is a freely formatted text desribing this file

Build command can be used to define a specific build command and flags for this file. Flags can either replace default flags (e.g. those specified for this file type) in the build script or be appended to them. Target name specifies a path to the file that is derived from this file when build process is run.
