Bus interface editor

Bus interfaces editor is used to edit the details of a single bus interface. Bus interface is used to group ports together to form an interface that i.e. fulfills requirements for a bus protocol. The editor contains two tabs: the general tab which is used to set the general settings of the interface, and the port maps tab which is used to group ports to the interface.

Bus interface has a mandatory name and textual description.
Bus definition and abstraction definition contain VLNV-references to IP-XACT documents that are associated with this interface. They define the qualities that this bus interface must meet. Bus definition is actually very small. The abstraction definition defines the logical signals and port maps define how they are connected to component's ports.

Interface mode combo box has 7 choices: 'direct'/'mirrored' 'master', 'slave', and 'system', as well as 'monitor'. Settings in the bottom are interface mode-specific and change when mode changes. For example, master and mirroredMaster have address space settings, whereas slave has memory map and bridge settings.

General settings include addressable unit size which defines how many bits are included in the least addressable unit of the bus. The default setting is byte addressable (8 bits). Endianness indicates whether the interface is 'big-endian' or 'little-endian' (default). Bit steering='on' implies that the interface is able to dynamically align data on different byte channels in case of addressable interfaces. The default setting when the bit steering is not set is off. Connection required indicates that when instantiated in a design, this interface must be connected to some other interface and cannot be left unconnected.

Parameters are optional and can specify any parameter data value(s) for this bus interface.
