Non-hierarchical (flat) view editor

Flat view editor is used to edit the non-hierarchical views of a component. The top part of the editor is identical with hierarchical view. User may change the view type and then the bottom part changes accordingly.

View has name and textual description. Display name can be more detailed or user-friendly than name, but usually only few words.

Environment identifiers in mandatory textual information about how this view deployed in tool environment. Language indicates this view may be compatible with a particular tool, but only if that language is supported in that tool, e.g., different versions of some simulators may support two or more languages. Tool indicates that this view contains tool-information, e.g., HDL models that use simulator-specific extensions. The tool can have a generic value such as "*Simulation" and "*Synthesis"VendorSpecific can further qualify tool and language compatibility, e.g. if some appropriate simulator mode must be enabled.

View type can be changed and a non-hierarchical view has 5 additional optional settings.

  • Language specifies the HDL for the view, e.g. VHDL, Verilog, or SystemC. b>Strict denotes if language shall be strictly enforced. The default is false (unchecked).
  • Model name is an optional language-specific identifier. For example, this may be of form entity(architecture) if name-field of the VLNV is not exactly the same as the entity name in VHDL code, or this may be VHDL configuration name. For SystemC, this is the sc_module class name.
  • File set references lists the used file sets of the containing component, e.g. hdl_sources.
  • Default file build commands lists the commands and flags to compile the files of the listed file sets.
  • Parameters are optional details that describe the view for generator usage.
