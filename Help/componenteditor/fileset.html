File set editor

File set editor can be used to edit the details of a single file set and to add and remove files contained in the file set. File sets can be used to group files together to be referenced by other sections of a component.

Name, display name, and description of the file set are shown on top. Name is mandatory as it identifies the file set within a component. Display name is typically a few words providing a more detailed and/or user-friendly name for display. Description contains a textual description of the file set.

Files

Files group box lists the files contained in the file set. Each file has a name identifying the file. The file name is mandatory, and it is given when adding file.
File path is defined either as relative to the component's xml-file or as an absolute path. The file path is mandatory and it is given when adding files.
File type links files to their build command (see below). One file can have many types. File type is mandatory.
Description is a freely formatted text describing a single file.

Add Files button opens a dialog to select files on the disk to add to the file set. The order of files is maintained and can be changed by dragging rows. If the compilation order of files is important then the files should be listed in order needed by compilation.
If a file is in invalid state, i.e. missing a mandatory file type, then the file is displayed in red color.

The files box also contains a context menu (right mouse button) that has the following options:

  • Add new file to the file set (Add row)
  • Remove file from file set (Remove row)
  • Clear the selected cells
  • Copy the contents of the selected cells to clipboard
  • Paste the contents of the clipboard to the selected cells
  • Import a csv-file to the editor
  • Export the contents of the editor to a csv-file

Default build commands

Default build command can define command and flags to build files, e.g. running gcc for C files or vcom for VHDL. File type must macth the file table above. The defined command can either replace default flags or be appended to it. This table has a similar context menu as the files editor.

Group identifiers

Group identifier can be used to describe the function or purpose of the file with a single unbounded word. Group identifiers can contain several identifiers and the possible words are not limited. However the editor suggests the following options:

  • Diagnostics
  • Documentation
  • ProjectFiles
  • Simulation
  • sourceFiles

Dependent directories

Dependent directories can specify a freetext list of paths to directories on which this file set depends, e.g. directories of include files.
